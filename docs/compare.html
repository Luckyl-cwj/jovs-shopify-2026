这次补充的首屏优化

这里补的是“为什么会更快、更稳”，不只是对比结果本身。

首屏优先级治理
  • 关键样式先到，整包样式后到 首屏 Banner 结构样式内联，其余 CSS 改成 preload，浏览器会先把第一屏排出来。
  • 连接预热前置 增加 dns-prefetch 与 3 个 preconnect，优先建立到 jovs.com 和字体源的连接。
  • 视频只预热当前设备真正会播的版本 顶部 Banner 视频源先挂在 data-src，只给当前桌面端或移动端那支视频注入真实 source。
  • 首屏资源不强行 lazy 3 张首屏核心图保持 eager/high，避免“看得到位置，但资源优先级被压后”的反效果。

性能层补充

  • 首屏 Banner 样式已经独立抽离 不是继续等 home.css 全量下载完，而是先把首屏必须用到的样式直接内联在页面头部。
  • 8 个非首屏模块改成按视区初始化 Awards Logo、Add Text Card、Prohibit Select、Add Blog 等模块接近视区时再初始化。
  • 首屏媒体只做必要预热 顶部视频用 metadata 预加载，真实视频源延后注入；首屏图片才保留 eager/high。

稳定性补充

  • swiper-prohibit-select 初始高度补了二次刷新 初始化后和图片加载完成后都会主动 update()，减少“刚进来高度不对，滑两次才恢复”的问题。
  • 轮播导航、进度条、resize 逻辑统一收口 页面里多个 Swiper 不再各写一套同步逻辑，进度条与按钮状态现在由统一方法更新。
  • 不需要轮播的端直接销毁实例 这能减少无意义的监听和布局干扰，也避免移动端 / 桌面端布局互相污染。

语义与可维护性补充

  • CTA 和轮播控件补齐语义 Shop now、Learn more、View all 这类通用文案现在都有更具体的访问语义。
  • 图片 alt 与装饰图空 alt 更规范 不是简单“全部都有 alt”，而是只让真正装饰性的图片保留空 alt，减少语义噪音。
  • 后续更适合继续做 section 化 Swiper 控件、进度条、断点策略和公共样式都已经收敛，后面接回 Theme Editor 的成本会更低。

当前线上遗留问题

这部分不是重构版新增的问题，而是当前线上版本里比较明显、且本次重构已经刻意规避的交互与样式风险。

保留截图方便沟通
  • 交互偶发无法收起 当前线上交互存在状态回收不稳定的问题，重构版通过统一按钮状态和键盘控制逻辑降低了这类风险。
  • 图片固定高度导致拉伸失真 这类问题本质上是尺寸策略和容器比例不稳定，重构版已经尽量通过显式尺寸和比例约束减少这类异常。
  • 交互激活状态不够明确 重构版把导航、进度和 active 状态同步收口，就是为了减少“能点但看不出当前状态”的体验问题。

一句话结论

这次首页重构的价值，不只是把页面从 hovs.html 迁到 home.html， 而是把首屏样式、媒体优先级、Swiper 初始化时机和交互语义全部重新收口，让页面在 Shopify 体系里更轻、更稳，也更适合长期运营。

如果要对外解释，这一页最值得强调的补充点就是： 首屏关键样式已经抽离出来，非首屏轮播不再一次性全部启动，首屏资源和非首屏资源的优先级被明确分开了。
